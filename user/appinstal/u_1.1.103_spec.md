## 動作条件

最初にWindowsにDirectCloud ドライブをインストールする場合、必ず管理者権限でインストールする必要があります。  

同じPCで、ほかのWindowsユーザーがDirectCloud ドライブを使用している場合、DirectCloud ドライブをインストールする際の動作は、以下のようになります。  

### Windowsの管理者権限を持つユーザーがインストールを実行した場合

| DirectCloud ドライブの種類 | 動作 |
| --- | --- |
| 通常版（32bit、64bit） | 「ほかのユーザーがDirectCloud ドライブを使用しています。DirectCloud ドライブを強制終了させたあと、インストールしてもよろしいですか？」という画面が表示されます。<br>「OK」をクリックすると、ほかのユーザーが使用しているDirectCloud ドライブは強制終了され、インストールが実行されます。 |
| サイレントインストール版（32bit、64bit） | ほかのユーザーが使用しているDirectCloud ドライブは自動的に終了し、インストールが実行されます。 |

### Windowsの管理者権限を持たないユーザーがインストールを実行した場合

| DirectCloud ドライブの種類 | 動作 |
| --- | --- |
| 通常版（32bit、64bit） | 管理者のパスワード入力画面が表示されます。<br>パスワードを入力して「OK」をクリックすると、ほかのユーザーが使用しているDirectCloud ドライブは強制終了され、インストールが実行されます。 |
| サイレントインストール版（32bit、64bit） |  |

32bit版がインストールされている状態で64bit版のインストーラーを実行すると、「インストールができません」というメッセージが表示され、インストールが強制終了します。その場合、32bit版をアンインストールしてから、64bit版をインストールしてください。  

IT資産管理ツールの配布機能を利用しDirectCloud ドライブのサイレントインストーラーを配布することで、Windowsの管理者権限がないユーザーの端末にDirectCloud ドライブをインストールすることができます。  

通常のDirectCloud ドライブをアンインストールせずにサイレントインストール版をインストールすると、自動アップデート機能が残ってしまう可能性があります。必ず、現在のDirectCloud ドライブをアンインストールし、PCを再起動してから、サイレントインストール版をインストールしてください。

## マルチアカウントモードの制約事項について

同じWindowsにサインインしている複数のユーザーがDirectCloud ドライブにアクセスし、ほかのユーザーの影響を受けない独立した環境でDirectCloud ドライブを利用できるようにするには、DirectCloud ドライブのインストール時に、「アプリケーションモード」で「マルチアカウントモード」を選択する必要があります。  

マルチアカウントモードでは「ユーザー権限」でドライブ文字が割り当てられますが、Windowsでは基本的に管理者権限でのドライブ文字の割り当てが推奨されているため、以下の制約が生じます。  

### 「管理者権限」でDirectCloud ドライブを実行する場合

「マルチアカウントモード」では、「ユーザー権限」でドライブがマウントされます。  
DirectCloud ドライブのアプリケーションは「ユーザー権限」で実行される必要があるため、プログラムのメニューなどで「管理者権限で実行」を選択してDirectCloud ドライブを起動すると、権限の不一致（権限の衝突）によって、ドライブは正常にマウントされません。  

### 「管理者権限」で実行されるプログラムでDirectCloud ドライブにアクセスする場合

Windowsのプログラムは一般的に「ユーザー権限」で実行されますが、一部「管理者権限」を必要とするプログラムもあります。  
「管理者権限」で実行されるプログラムは、「ユーザー権限」でマウントされたDirectCloud ドライブにアクセスする際に、権限の不一致（権限の衝突）が原因でファイルに正常にアクセスできない可能性があります。  

### 「管理者権限」が必要なフォルダへ移動またはコピーする場合

「ユーザー権限」で割り当てられたDirectCloud ドライブのファイルまたはフォルダを、「管理者権限」が必要なフォルダ（Cドライブなど）に移動またはコピーすると、権限の不一致（権限の衝突）でエラーが発生します。  

「DirectCloud ドライブセットアップ」画面でマルチアカウントモードを選択できるのは、サイレントインストール用を除く、Windows版DirectCloud ドライブのインストーラーのみとなります。  
以下のインストーラーでは、自動的に「シングルアカウントモード」が選択された状態になります。  

- Windows版 DirectCloud ドライブ サイレントインストール用  
- Windows版 DirectCloud ドライブ（DirectCloud-SHIELD IRM 統合版）※利用条件  
- Windows版 DirectCloud ドライブ（DirectCloud-SHIELD DLP 統合版）※利用条件  

インストールするWindows版DirectCloud ドライブのバージョンによっては、「.Net Framework」と「Microsoft Edge WebView2 Runtime」が必要です。  
追加インストールが必要かどうかは、OSによって異なります。  

| OS | 追加インストール |
| --- | --- |
| Windows 10／11 | 標準でインストールされているため、追加インストールする必要はありません。 |
| Windows Server 2019／2022 | 「.Net Framework」は標準でインストールされていますが、「Microsoft Edge WebView2 Runtime」はインストールされていません。 |
| Windows Server 2016 | 「.Net Framework」「Microsoft Edge WebView2 Runtime」ともに標準ではインストールされていません。 |

「Microsoft Edge WebView2 Runtime」は、Microsoft Officeをインストールする際に、自動的にインストールされるため、環境によっては追加インストールが必要ない場合もあります。  
追加インストールが必要な場合は、DirectCloud ドライブのインストール時に自動的にインストールされます。そのため、通常のインストールより時間がかかる場合があります。  

「.Net Framework 4.8」「Microsoft Edge WebView2 Runtime」の追加インストールには、インターネットに接続できる環境が必要です。

## 契約プランに関する制限・補足事項

契約プランがビジネス以上で、DirectCloudの有料オプションである「DirectCloud-SHIELD IRM」を契約している場合、DirectCloud ドライブ（DirectCloud-SHIELD IRM 統合版）を利用することができます。  

契約プランがビジネス以上で、DirectCloudの有料オプションである「DirectCloud-SHIELD DLP」を契約している場合、DirectCloud ドライブ（DirectCloud-SHIELD DLP 統合版）を利用することができます。  

---

## 仕様に関する制限・補足事項

DirectCloud ドライブのサイレントインストーラーには自動アップデート機能がないため、サイレントインストール版をインストールした端末には、アップデート通知は表示されません。  

ユーザーがログイン可能なアプリケーションを制限する方法で、「DirectCloud ドライブ」が許可されていない場合は、DirectCloud ドライブにログインできません。  

DirectCloud ドライブで可能な操作については、アクセスレベルの種類とDirectCloud ドライブで可能な操作を参照してください。
